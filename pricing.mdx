Velobase uses a transaction-based pricing model: you are only charged for successful `Deduct` and `Consume` operations. Deposits, freezes, unfreezes, and balance queries never count against your usage — they are always free.

## Billable vs. free operations

| Operation | Billable? |
|---|---|
| Deposit | No |
| Freeze | No |
| Consume | **Yes** |
| Unfreeze | No |
| Deduct | **Yes** |
| Get balance | No |

Only the two operations that finalize a credit deduction from a customer's balance count as ledger transactions.

## Free plan hard limit

The Free plan allows up to **50,000 ledger transactions per month**. Once you reach that limit, `Deduct` and `Consume` calls return HTTP `429`. Deposits, freezes, unfreezes, and balance queries continue to work normally.

```json
{
  "error": {
    "type": "too_many_requests",
    "code": "usage_limit_exceeded",
    "message": "monthly transaction limit reached — upgrade your plan"
  }
}
```

To avoid hard limits, upgrade to a Pro plan before you approach 50,000 transactions in a calendar month.

## Billing cycle

- Each organization has a single billing subscription.
- The billing period is calendar-monthly, starting from the date your subscription was created.
- Usage is tracked per organization across all projects. The dashboard shows a per-project breakdown so you can identify which projects are consuming the most transactions.
- Pro overage charges are calculated and billed at the end of each cycle.
